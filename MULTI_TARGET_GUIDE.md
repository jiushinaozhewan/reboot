# 多目标控制功能使用指南

## 🎉 新功能概览

现在客户端支持同时管理和控制多个目标设备（最多200个），可以批量执行关机、重启、开机等操作。

## 📋 功能特性

### ✅ 核心功能
- **动态目标数量**：支持 1-200 个目标设备
- **批量操作**：可以同时对多个目标执行命令
- **实时状态显示**：每个目标独立显示连接状态和MAC地址
- **配置持久化**：自动保存所有目标配置
- **可滚动列表**：固定高度的可滚动表格，支持大量设备管理
- **全选功能**：快速选择/取消选择所有目标

## 📖 使用步骤

### 1️⃣ 设置目标数量
1. 在顶部 **目标数量** 输入框输入数字（1-200）
2. 点击 **确定** 按钮生成对应数量的目标行

### 2️⃣ 配置目标信息
每个目标需要配置：
- **别名**：自定义名称（如"服务器1"）
- **IP地址**：目标设备IP（如 192.168.1.10）
- **端口**：Agent监听端口（默认 7890）

### 3️⃣ 设置全局密钥
在顶部 **全局密钥** 输入框输入64位十六进制PSK密钥（所有目标共享）

### 4️⃣ 保存配置
点击 **保存配置** 按钮，将当前所有目标配置保存到文件

### 5️⃣ 选择目标
- 勾选每个目标前的 **复选框** 来选择要操作的设备
- 点击表头的 **全选** 复选框可以快速选择/取消所有目标

### 6️⃣ 测试连接
1. 勾选需要连接的目标
2. 点击底部 **测试连接** 按钮
3. 等待连接完成，观察每行的状态变化：
   - `⟳ 连接中` - 正在连接
   - `● 已连接` - 连接成功（绿色）
   - `○ 断开` - 未连接（灰色）
   - `⚠ 错误` - 连接失败（红色）
4. 连接成功后，MAC地址会自动显示

### 7️⃣ 执行批量操作

#### 🔴 关机
1. 勾选需要关机的目标（必须是已连接状态）
2. 点击 **关机** 按钮
3. 确认对话框显示将要关机的目标数量
4. 点击 **确定** 执行（30秒后生效）

#### 🔵 重启
1. 勾选需要重启的目标（必须是已连接状态）
2. 点击 **重启** 按钮
3. 确认对话框显示将要重启的目标数量
4. 点击 **确定** 执行（30秒后生效）

#### 🟢 开机 (WoL)
1. 勾选需要唤醒的目标（必须已保存MAC地址）
2. 点击 **开机 (WoL)** 按钮
3. 系统会发送WoL魔术包到所选目标

## 🔍 状态说明

### 连接状态图标
- `● 已连接` - 绿色，设备已成功连接
- `○ 断开` - 灰色，设备未连接
- `⟳ 连接中` - 蓝色，正在建立连接
- `⚠ 错误` - 红色，连接或操作失败

### MAC地址显示
- `AA:BB:CC:DD:EE:FF` - 已获取的MAC地址
- `未获取` - 尚未连接或获取失败

### 底部状态栏
显示当前操作的结果：
- `已选中: 3/10` - 当前选中3个，共10个目标
- `连接完成: 3 成功, 1 失败` - 批量操作结果统计
- `操作完成: 5 个成功` - 所有操作成功

## ⚙️ 高级功能

### 并发控制
- 批量连接时，系统会并发处理所有选中的目标
- 每个目标独立执行，互不影响
- 失败的目标会显示错误信息，不影响其他目标

### 配置文件位置
配置自动保存在：
```
Windows: C:\Users\<用户名>\AppData\Roaming\reboot-client\config.toml
```

### 配置文件格式示例
```toml
psk_hex = "0123456789abcdef..."
target_count = 10

[[targets]]
alias = "服务器1"
ip = "192.168.1.10"
port = "7890"
mac = "AA:BB:CC:DD:EE:FF"

[[targets]]
alias = "服务器2"
ip = "192.168.1.11"
port = "7890"
```

## 💡 使用技巧

1. **快速批量设置**：复制粘贴IP地址，使用Tab键快速切换输入框
2. **分批操作**：可以先连接部分目标，测试成功后再连接其他目标
3. **保存配置**：每次修改目标信息后记得点击"保存配置"
4. **MAC地址**：首次连接成功后会自动保存MAC，之后可以直接使用WoL唤醒
5. **滚动查看**：使用鼠标滚轮或拖动右侧滚动条查看更多目标

## ⚠️ 注意事项

1. **目标数量限制**：最多支持200个目标，建议根据实际需求设置
2. **密钥统一**：所有目标使用相同的PSK密钥
3. **连接状态**：关机/重启操作只对"已连接"状态的目标有效
4. **MAC地址要求**：WoL开机需要目标设备支持并启用了Wake-on-LAN功能
5. **网络要求**：确保客户端与所有目标设备网络互通

## 🐛 常见问题

### Q: 为什么某些目标连接失败？
A: 检查以下项：
- IP地址和端口是否正确
- 目标Agent是否正在运行
- 网络是否通畅（可以先ping测试）
- PSK密钥是否与Agent配置一致

### Q: MAC地址显示"未获取"？
A: 可能原因：
- 尚未成功连接过该目标
- Agent获取MAC地址失败
- 需要重新测试连接

### Q: 可以对不同目标使用不同密钥吗？
A: 当前版本不支持，所有目标共享同一个全局PSK密钥

### Q: 如何快速清空所有目标？
A: 将目标数量设为较小值（如1），点击确定，多余的目标会被删除
